SQL for Data Analysis ‚Äî COVID-19 Case Study (Italy)

As part of the Data Analytics Bootcamp, I completed a rigorous 7-day intensive course in SQL for Data Analysis, focused on querying, transforming, and drawing insights from large-scale public health data using Google BigQuery. The course emphasized real-world application of SQL by analyzing COVID-19 case data from Italy, spanning national, regional, and provincial levels. With over 300,000 rows of data across multiple tables, this project provided hands-on experience in navigating relational databases, constructing complex queries, performing data manipulation, and generating actionable insights.

The learning curve was steep due to two key challenges: a lack of prior SQL experience and limited domain knowledge in epidemiological data. To overcome the SQL barrier, I practiced by writing queries on real-world datasets, gradually building from basic SELECT statements to more advanced operations like JOINS and subqueries. I used Google BigQuery as my sandbox environment and developed a habit of reverse-engineering existing queries to understand the logic behind each clause. I also challenged myself to write the same output with different approaches (e.g., subquery vs. CTE).

To bridge the domain knowledge gap, I began by reviewing trusted sources like the WHO and Our World in Data to understand key public health metrics such as positivity rates and hospitalization trends. I created a data dictionary not just as documentation, but as a thinking tool, annotating it with definitions, assumptions, and clarifying questions. This helped me translate field names like new_tested_positive into meaningful indicators. I also compared patterns across provinces and time periods to better grasp the logic of the data and recognize anomalies. These strategies enabled me to approach the technical exercises and final SQL exam with confidence and contextual understanding.

The project was structured around solving real-world analytical problems ‚Äî such as identifying regional case surges, calculating positivity and fatality rates, and analyzing hospitalization patterns over time. I used a wide range of SQL techniques, including aggregations, conditional logic, data manipulation, multi-table JOINs, and subqueries, to answer progressively complex questions. The course culminated in a hands-on technical exam, where I independently solved a set of case-based problems using SQL to extract, transform, and interpret meaningful insights from the data.

Sample Analytical Problems Solved:

  • Identified top contributing provinces and regions to national COVID-19 totals on selected dates.
  • Calculated daily positivity and fatality rates using dynamic expressions and NULL handling.
  • Analyzed hospitalization trends by region and over time to identify hotspots and anomalies.
  • Applied CASE WHEN logic to classify regions by severity of case increases.
  • Used subqueries to compare regional rates against national averages.
  • Grouped and filtered data using complex multi-date conditions for seasonal and monthly comparisons.

Key Skills and Tools Strengthened:

  • SQL Querying: Mastered SELECT, WHERE, GROUP BY, HAVING, ORDER BY, LIMIT, JOINs (inner/left), CASE WHEN logic, and nested subqueries to answer analytical questions.
  • Data Manipulation: Transformed raw datasets into meaningful formats using aggregations, calculated fields, conditional logic, and structured filtering to support analysis.
  • Relational Data Analysis: Explored hierarchical relationships between provinces, regions, and national aggregates to assess contributions and trends at multiple levels.
  • BigQuery Proficiency: Used Google BigQuery to query datasets with 300,000+ rows; applied timestamp functions, best practices in code formatting, aliases, and performance optimization.
  • Data Wrangling & Validation: Constructed a working data dictionary for undocumented datasets; identified schema inconsistencies and handled missing values using logical assumptions and data proxies.
  • Applied Analytics: Computed hospitalization, positivity, and fatality rates; tracked regional trends and performed date-based filtering (daily, seasonal, monthly).
  • Critical Thinking: Broke down multi-step analytical challenges into SQL-driven solutions, using conditional logic and cross-table comparisons to inform decision-making.
  • Analytical Storytelling: Presented findings with context and clarity, framing outputs as business-like answers to questions on public health trends and data reliability.

This project demonstrated end-to-end proficiency in SQL and BigQuery, from raw data exploration to high-level insight generation, while also showcasing adaptability in overcoming technical and domain-specific challenges in a fast-paced, real-world setting.
